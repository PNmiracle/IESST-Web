IESST-Web 阿里云部署指南

把 GitHub 项目 PNmiracle/IESST-Web 部署到阿里云 ECS，并绑定域名 iesst.xyz。

0. 总路线

用户访问 https://iesst.xyz
        ↓
阿里云 DNS 解析到 ECS 公网 IP
        ↓
Nginx 监听 80/443
        ↓
反向代理到 Docker 里的 Spring Boot:8080
        ↓
Spring Boot + Vue 静态页面 + MySQL + OSS

你的项目已经有生产部署文件，主要用：

Dockerfile
docker-compose.prod.yml
.env

1. 先确认是否需要备案

如果你的 ECS 买在中国内地，比如杭州、上海、北京、深圳，iesst.xyz 必须完成 ICP 备案后才能正式访问。

如果你想先快速上线测试，建议 ECS 选香港、新加坡、日本、美国等非中国内地区域，可以先不走中国内地备案流程。

  • 阿里云备案入口
  • 备案控制台

2. 创建 ECS 服务器

进入 阿里云 ECS 控制台，建议配置：

地域：香港 / 新加坡 / 中国香港，想免备案就选非中国内地
系统：Ubuntu 22.04 LTS 或 Ubuntu 24.04 LTS
规格：最低 2 核 4G，推荐 2 核 8G
公网 IP：需要
带宽：3M 起步，正式可 5M+
磁盘：40GB 起步，推荐 80GB

你的生产 Compose 会跑 app + mysql + clamav，ClamAV 比较吃内存，所以 2G 机器不太舒服。

  • 阿里云 ECS 控制台
  • 阿里云 ECS 文档中心

3. 配置安全组

进入 ECS 实例详情：

ECS 控制台 -> 实例 -> 你的实例 -> 安全组 -> 配置规则

添加入方向规则：

协议类型：TCP
端口范围：22
授权对象：你的本机公网 IP/32
用途：Workbench/SSH 登录
协议类型：TCP
端口范围：80
授权对象：0.0.0.0/0
用途：HTTP，申请 HTTPS 证书也需要
协议类型：TCP
端口范围：443
授权对象：0.0.0.0/0
用途：HTTPS
不要开放 8080、3306、3310，这些只给服务器内部用。
  • 安全组官方文档
  • 安全组控制台

4. 用 Workbench 连接 ECS

ECS 控制台 -> 实例 -> 远程连接 -> Workbench 远程连接

登录用户通常是 root。如果你创建实例时设置的是普通用户，就用对应用户名。

  • Workbench 连接 Linux 实例文档
  • ECS 控制台

5. 安装基础软件

Workbench 打开终端后执行：

sudo apt update
sudo apt install -y git curl vim nginx certbot python3-certbot-nginx

安装 Docker：

sudo apt install -y ca-certificates gnupg lsb-release docker.io docker-compose-plugin
sudo systemctl enable --now docker

验证：

git --version
docker --version
docker compose version
nginx -v
  • Docker Ubuntu 安装文档
  • Docker Compose Linux 安装文档
  • Certbot Nginx 文档

6. 创建 OSS Bucket

进入 OSS 控制台，建议：

Bucket 名称：iesst-manuscripts-你的后缀
地域：尽量和 ECS 同地域或近一点
读写权限：私有
版本控制：可选开启
服务端加密：建议开启

你的项目生产配置里 MANUSCRIPT_STORAGE_BACKEND=oss，所以投稿文件会走 OSS。

  • OSS 控制台
  • OSS 官方文档

7. 创建 RAM 用户和 AccessKey

进入 RAM 控制台，建议流程：

身份管理 -> 用户 -> 创建用户
用户名：iesst-oss-uploader
访问方式：OpenAPI 调用访问

然后给这个 RAM 用户授权 OSS 权限。最简单可先用 AliyunOSSFullAccess，正式生产更推荐只给指定 Bucket 的最小权限。

创建 AccessKey：

RAM 用户详情 -> 认证管理 / AccessKey -> 创建 AccessKey

保存 AccessKey ID 和 AccessKey Secret。注意：AccessKey Secret 只显示一次。

  • RAM 控制台
  • 创建 AccessKey 官方文档

8. 拉取你的 GitHub 项目

cd /opt
sudo git clone https://github.com/PNmiracle/IESST-Web.git iesst-web
sudo chown -R $USER:$USER /opt/iesst-web
cd /opt/iesst-web

确认文件存在：

ls
ls docker-compose.prod.yml Dockerfile .env.example
  • 你的 GitHub 仓库
  • GitHub Docs

9. 配置生产环境 .env

cd /opt/iesst-web
cp .env.example .env
vim .env

推荐填成这样：

APP_PORT=8080
MYSQL_PORT=3307

DB_PASSWORD=这里换成数据库强密码
MYSQL_ROOT_PASSWORD=这里换成MySQL-root强密码

ADMIN_USERNAME=admin
ADMIN_PASSWORD=这里换成后台强密码
ADMIN_DISPLAY_NAME=IESST 管理员

PII_ENCRYPTION_KEY=这里填openssl生成的第一个key
PII_LOOKUP_KEY=这里填openssl生成的第二个key

OSS_ENDPOINT=https://oss-cn-hongkong.aliyuncs.com
OSS_BUCKET=你的OSS-Bucket名
OSS_ACCESS_KEY_ID=你的RAM-AccessKey-ID
OSS_ACCESS_KEY_SECRET=你的RAM-AccessKey-Secret
OSS_MANUSCRIPT_PREFIX=manuscripts

生成两个 PII Key：

openssl rand -base64 32
openssl rand -base64 32

如果你的 OSS 在杭州，OSS_ENDPOINT 用 https://oss-cn-hangzhou.aliyuncs.com；如果在香港，常用 https://oss-cn-hongkong.aliyuncs.com。

10. 启动生产服务

cd /opt/iesst-web
docker compose -f docker-compose.prod.yml up -d --build

查看容器：

docker compose -f docker-compose.prod.yml ps

看后端日志：

docker logs -f iesst-app

验证健康检查：

curl http://127.0.0.1:8080/actuator/health

看到这个就对了：

{"status":"UP"}

如果 ClamAV 第一次启动慢，等 2-5 分钟再查：

docker logs -f iesst-clamav
  • Docker Compose 文档

11. 配置域名 DNS 解析

进入 阿里云 DNS 控制台，找到 iesst.xyz，添加两条 A 记录：

记录类型：A
主机记录：@
记录值：你的 ECS 公网 IP
TTL：默认
记录类型：A
主机记录：www
记录值：你的 ECS 公网 IP
TTL：默认

本地验证：

dig iesst.xyz
dig www.iesst.xyz

如果没有 dig：

ping iesst.xyz
  • 云解析 DNS 控制台
  • 添加解析记录官方文档

12. 配置 Nginx 反向代理

sudo vim /etc/nginx/sites-available/iesst.xyz

写入：

server {
    listen 80;
    server_name iesst.xyz www.iesst.xyz;

    client_max_body_size 25m;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

启用配置：

sudo ln -s /etc/nginx/sites-available/iesst.xyz /etc/nginx/sites-enabled/iesst.xyz
sudo nginx -t
sudo systemctl reload nginx

验证：

curl -I http://iesst.xyz
  • Nginx 官方文档
  • Certbot Nginx 前置要求

13. 申请 HTTPS 证书

确认这两个地址已经能通过 HTTP 打开：

http://iesst.xyz
http://www.iesst.xyz

然后执行：

sudo certbot --nginx -d iesst.xyz -d www.iesst.xyz

按提示输入邮箱、同意条款，选择自动跳转 HTTPS。

验证续期：

sudo certbot renew --dry-run
  • Certbot Nginx 官方指南
  • Let’s Encrypt 文档

14. 最终访问

https://iesst.xyz
https://iesst.xyz/admin

后台账号密码是 .env 里这两个：

ADMIN_USERNAME=admin
ADMIN_PASSWORD=你设置的后台密码

15. 每次更新 GitHub 后，部署环境怎么同步

以后你的本地代码改完并上传到 GitHub 后，ECS 服务器不会自动更新。你需要登录阿里云 Workbench，在部署目录里强制同步 GitHub 的 main，然后重建 Docker。

从现在开始，线上部署统一只用 main 分支。其他开发分支合并进 main 后再部署，服务器不再直接拉开发分支。

本地电脑先把代码推到 GitHub：

git status
git add .
git commit -m "说明这次更新了什么"
git push origin main

然后登录阿里云 Workbench，进入服务器上的项目目录：

cd /opt/iesst-web

部署前先看当前服务器版本，方便出问题时回滚：

git branch --show-current
git log --oneline -5

建议把当前线上版本的 commit id 复制保存一下，例如 a1b2c3d。

如果这次更新涉及数据库结构、投稿、用户数据或后台重要内容，先备份 MySQL：

mkdir -p backups
docker exec iesst-mysql-prod mysqldump -uroot -p"$MYSQL_ROOT_PASSWORD" iesst > backups/iesst-$(date +%Y%m%d-%H%M%S).sql
如果上面命令提示 MYSQL_ROOT_PASSWORD 为空，可以先执行 set -a && source .env && set +a，再重新运行备份命令。

强制同步 GitHub 最新 main：

git fetch origin
git reset --hard origin/main
这会让服务器项目目录完全等于 GitHub 的 main，避免 git pull 遇到 divergent branches。服务器只负责运行，不在服务器上改代码。

重新构建并启动生产容器：

docker compose -f docker-compose.prod.yml up -d --build

这一步会重新构建前端和后端镜像，然后替换旧的应用容器。MySQL 数据在 Docker volume 里，正常不会被删除。

检查容器状态：

docker compose -f docker-compose.prod.yml ps

看后端启动日志：

docker logs -f iesst-app

看到类似 Started IesstDemoApplication，基本说明后端启动成功。

验证健康检查：

curl http://127.0.0.1:8080/actuator/health

期望返回：

{"status":"UP"}

验证公网访问：

curl -I https://iesst.xyz
curl -I https://iesst.xyz/admin

期望看到 HTTP/2 200、HTTP/1.1 200 或正常的登录页响应。

如果只是想重启应用，不拉新代码：

cd /opt/iesst-web
docker compose -f docker-compose.prod.yml restart app

如果部署后发现新版本有问题，可以先回滚到之前保存的 commit id：

cd /opt/iesst-web
git checkout a1b2c3d
docker compose -f docker-compose.prod.yml up -d --build

确认恢复后，如果想回到主分支继续后续更新：

git checkout main
git fetch origin
git reset --hard origin/main

如果要停止服务：

cd /opt/iesst-web
docker compose -f docker-compose.prod.yml down
不要随便执行 docker compose -f docker-compose.prod.yml down -v，因为 -v 会删除数据库 volume，线上数据会丢。

最常用的更新命令可以记这一组：

cd /opt/iesst-web
git fetch origin
git reset --hard origin/main
docker compose -f docker-compose.prod.yml up -d --build
docker compose -f docker-compose.prod.yml ps
curl http://127.0.0.1:8080/actuator/health
