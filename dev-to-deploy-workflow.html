IESST-Web 开发到部署完整流程

这份文档讲清楚：你在本地开发环境改代码，推送到 GitHub，然后在阿里云 Workbench 更新线上部署环境。线上统一只用 main 分支。

本地开发环境

你的电脑。访问 http://127.0.0.1:8080 或 http://127.0.0.1:5173。

GitHub

代码仓库。最终版本统一推到 main。

阿里云部署环境

ECS 服务器。通过 Workbench 登录，在 /opt/iesst-web 更新并重建 Docker。

最终规则：以后不要再拉开发分支部署。线上服务器只同步 origin/main。

0. 先理解 127.0.0.1

127.0.0.1 永远代表“当前这台机器自己”。

你电脑浏览器里的 127.0.0.1:8080
= 你的 Mac 本机开发环境

阿里云服务器命令行里的 127.0.0.1:8080
= ECS 服务器内部的 Spring Boot

外部用户访问线上网站
= https://iesst.xyz

线上链路是：

浏览器 -> https://iesst.xyz -> Nginx 443 -> http://127.0.0.1:8080 -> Docker 里的 Spring Boot

1. 本地开发前同步 main

在你电脑本地项目目录执行：

cd /Users/chengxinzhi/Downloads/IESST-Academic-Platform
git switch main
git fetch origin
git reset --hard origin/main

这样保证你本地开发是从 GitHub 最新 main 开始。

git reset --hard 会丢弃本地未提交改动。执行前如果有正在改的内容，先提交或备份。

2. 本地启动开发环境

如果只想跑单体后端页面：

./scripts/start-mysql.sh
./scripts/start-server.sh

打开：

http://127.0.0.1:8080

如果要前端热更新：

./scripts/start-mysql.sh
./scripts/start-server.sh
./scripts/start-client.sh

打开：

http://127.0.0.1:5173

3. 本地改代码并测试

修改前端、后端或配置后，至少做这几项检查：

curl http://127.0.0.1:8080/actuator/health
curl -i http://127.0.0.1:8080/api/public/journals

后端测试：

./.tools/apache-maven-3.9.9/bin/mvn -f server/pom.xml test

前端构建：

cd client
npm run build

4. 提交并推送到 GitHub main

确认改动：

git status

提交：

git add .
git commit -m "说明这次更新了什么"
git push origin main

推送成功后，GitHub 的 main 就是你准备部署的最新版本。

5. 登录阿里云 Workbench

进入：

阿里云 ECS 控制台 -> 实例 -> 远程连接 -> Workbench

登录服务器后进入线上项目目录：

cd /opt/iesst-web

6. 部署前看当前状态

git status
git branch --show-current
git log --oneline -5
docker compose -f docker-compose.prod.yml ps

正常情况下服务器在 main 分支。即使它不在，也可以用下一步强制同步回来。

7. Workbench 上同步 GitHub main

以后服务器更新统一用这两条，不用 git pull：

git fetch origin
git reset --hard origin/main

这样可以避免 divergent branches 分叉错误。服务器只负责运行，不在服务器上开发代码。

如果你曾经直接在服务器里改过代码，这一步会丢掉那些服务器本地改动。正确做法是：代码改动只在本地电脑完成，再推到 GitHub。

8. 重新构建并启动线上 Docker

docker compose -f docker-compose.prod.yml up -d --build

刚启动完时，iesst-app 可能显示 health: starting，这时访问域名可能短暂出现 502 Bad Gateway。等 30 秒到 2 分钟再检查。

9. 检查容器健康

docker compose -f docker-compose.prod.yml ps

理想状态：

iesst-app         Up ... (healthy)
iesst-mysql-prod  Up ... (healthy)
iesst-clamav      Up ... (healthy)

如果 iesst-app 一直不是 healthy，看日志：

docker logs --tail=150 iesst-app

10. 检查服务器内部后端

curl http://127.0.0.1:8080/actuator/health
curl -i http://127.0.0.1:8080/api/public/journals

健康检查应该返回：

{"status":"UP"}

API 应该包含：

Content-Type: application/json

11. 检查线上域名 API

curl -i https://iesst.xyz/api/public/journals

正确结果应该包含：

Content-Type: application/json

如果看到这个，说明 Nginx 又把 API 请求返回成前端页面了：

Content-Type: text/html
<!doctype html>

这时检查 Nginx 启用站点：

ls -l /etc/nginx/sites-enabled/

正常只保留 iesst.xyz。如果又出现旧配置：

sudo unlink /etc/nginx/sites-enabled/iesst
sudo unlink /etc/nginx/sites-enabled/certbot-temp
sudo nginx -t
sudo systemctl reload nginx

12. 浏览器最终检查

打开：

https://iesst.xyz
https://iesst.xyz/services/translation
https://iesst.xyz/EI
https://iesst.xyz/admin

如果看起来还是旧页面，强制刷新：

Mac：Command + Shift + R
Windows：Ctrl + F5

13. 如果线上还是旧版本

先确认服务器已经拿到 GitHub 最新提交：

git log --oneline -5

再无缓存重建：

docker compose -f docker-compose.prod.yml down
docker image rm iesst-academic-platform:latest
docker compose -f docker-compose.prod.yml build --no-cache
docker compose -f docker-compose.prod.yml up -d

然后重复健康检查和 API 检查。

14. 如果要回滚

查看最近版本：

git log --oneline -10

临时切回某个旧 commit：

git checkout 旧commitID
docker compose -f docker-compose.prod.yml up -d --build

恢复到最新 main：

git switch main
git fetch origin
git reset --hard origin/main
docker compose -f docker-compose.prod.yml up -d --build

15. 每次更新最短命令

本地电脑

git status
git add .
git commit -m "说明这次更新了什么"
git push origin main

阿里云 Workbench

cd /opt/iesst-web
git fetch origin
git reset --hard origin/main
docker compose -f docker-compose.prod.yml up -d --build
docker compose -f docker-compose.prod.yml ps
curl http://127.0.0.1:8080/actuator/health
curl -i https://iesst.xyz/api/public/journals

16. 不要做的事

  • 不要在服务器上直接改代码然后再 git pull。
  • 不要把 .env 提交到 GitHub。
  • 不要随便执行 docker compose -f docker-compose.prod.yml down -v，它会删除数据库 volume。
  • 不要用浏览器访问自己电脑的 127.0.0.1 来判断线上是否成功。线上看 https://iesst.xyz。
