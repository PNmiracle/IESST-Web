# IESST 学术服务平台：导师演示版启动说明

这个压缩包已经包含完整的 Vue 前端、Spring Boot 后端、MySQL 数据库结构、Flyway 迁移脚本、网站图片和演示数据。推荐使用 Docker Desktop 启动，不需要单独安装 Node.js、Java、Maven 或 MySQL。

> 本包仅用于本地演示。内置账号和密钥是公开的演示凭据，不能直接用于互联网生产环境。

## 1. 电脑要求

- Windows 10/11（64 位）或 macOS 12 及以上
- 至少 4 GB 可用内存、5 GB 可用磁盘
- 首次启动需要联网下载 Docker 镜像和项目依赖
- 安装并启动 [Docker Desktop](https://www.docker.com/products/docker-desktop/)

确认 Docker Desktop 左下角显示 Engine running 后再继续。

## 2. 一键启动

### macOS

打开“终端”，进入解压后的项目目录。例如：

```bash
cd ~/Downloads/IESST-Mentor-Package
./scripts/mentor-start.sh
```

若 macOS 首次阻止脚本执行：

```bash
chmod +x scripts/mentor-*.sh
./scripts/mentor-start.sh
```

### Windows

双击根目录的 `启动导师演示版.bat`。也可以在 PowerShell 中运行：

```powershell
docker compose -f docker-compose.mentor.yml up -d --build
```

分享包已包含构建完成的前后端单体 JAR。首次启动一般需要 2-8 分钟，主要用于下载 Java 与 MySQL 镜像；后续启动会明显更快。

## 4. 推荐演示路径

1. 打开官网，浏览 SCI/EI 期刊、翻译润色、科学编辑和关于我们。
2. 点击“免费评估稿件”，填写演示信息并上传一个无敏感内容的 PDF/Word 文件。
3. 登录学生端，查看订单、流程进度和文件。
4. 登录管理后台，查看投稿、更新状态、维护官网内容。
5. 回到学生端刷新，检查订单阶段是否同步更新。

## 5. 停止、重新启动与清空数据

macOS/Linux 停止：

```bash
./scripts/mentor-stop.sh
```

Windows 停止：双击 `停止导师演示版.bat`。

普通停止不会删除数据库。下次执行启动命令，之前的演示操作仍然存在。

完全清空并恢复初始演示数据：

```bash
./scripts/mentor-reset.sh
./scripts/mentor-start.sh
```

Windows PowerShell 完全清空：

```powershell
docker compose -f docker-compose.mentor.yml down -v
docker compose -f docker-compose.mentor.yml up -d --build
```

## 6. 常见问题

### 8080 端口已被占用

macOS/Linux 使用其他端口：

```bash
IESST_PORT=8088 ./scripts/mentor-start.sh
```

Windows PowerShell：

```powershell
$env:IESST_PORT="8088"
docker compose -f docker-compose.mentor.yml up -d --build
```

然后访问 `http://127.0.0.1:8088/`。

### 页面暂时打不开

MySQL 首次初始化和后端迁移需要一点时间。查看容器状态：

```bash
docker compose -f docker-compose.mentor.yml ps
```

`app` 和 `mysql` 都显示 healthy 后再刷新页面。

### 出现 docker.io / auth.docker.io / timeout / DeadlineExceeded

这是 Docker Hub 镜像下载链路超时，不是项目代码错误。先测试：

```bash
docker pull eclipse-temurin:17-jre
docker pull mysql:8.4
```

若仍超时，请更换网络，或在 Docker Desktop 的 `Settings -> Resources -> Proxies` 中配置当前可用的 HTTP/HTTPS 代理，重启 Docker Desktop 后再运行启动脚本。学校/公司网络可能会限制 Docker Hub；使用手机热点也可快速判断是否属于网络限制。

### 查看错误日志

```bash
docker compose -f docker-compose.mentor.yml logs --tail=200 app
docker compose -f docker-compose.mentor.yml logs --tail=200 mysql
```

持续查看日志：

```bash
docker compose -f docker-compose.mentor.yml logs -f
```

按 `Ctrl+C` 只会退出日志查看，不会停止网站。

### 修改代码后没有变化

重新构建应用镜像：

```bash
docker compose -f docker-compose.mentor.yml up -d --build app
```

## 7. 架构与数据位置

```text
浏览器
  -> Spring Boot :8080（API + 已构建的 Vue 页面）
      -> MySQL 8.4（Docker 内部网络，不暴露到宿主机）
      -> Docker 私有卷（演示上传文件）
```

- 数据表定义和升级脚本：`server/src/main/resources/db/migration/`
- 前端源码：`client/src/`
- 后端源码：`server/src/main/java/`
- Docker 构建：`Dockerfile`
- 导师版编排：`docker-compose.mentor.yml`
- 数据库和上传内容保存在 Docker named volumes 中，不会写入源码目录。

## 8. 安全说明

- 分享包不包含真实 `.env`、真实数据库备份、真实手机号、生产密钥或私人稿件。
- 学生手机号在数据库中使用 AES-256-GCM 加密，并使用 HMAC 检索值登录。
- 学生与管理员密码使用 BCrypt 哈希保存，接口不会返回密码哈希。
- 演示版关闭 OSS 和病毒扫描，投稿文件只保存在本机 Docker 私有卷中。
- 若未来公开部署，必须更换全部凭据、启用 HTTPS、私有 OSS、恶意文件扫描和生产 Profile。

## 9. 不使用 Docker 的开发启动（可选）

本地需安装 Java 17、Maven 3.9、Node.js 20 和 MySQL 8。按照根目录 `README.md` 的“前后端分离开发方式”执行。仅评审和演示时不推荐此方式。
